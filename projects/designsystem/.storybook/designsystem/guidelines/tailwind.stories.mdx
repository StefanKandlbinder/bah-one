# TailwindCSS

## Introduction
[Tailwind](https://tailwindcss.com/docs) provides a very practicable way,
to automatically generate utility classes, based on a fully customizable json config file.
They are very similiar to the classes we are alreading using in our desktop application,
like `l-mt0` which represents `margin-top: 0` -
the tailwind couterpart e.g. is `m-0`, see
[tailwind margin utility classes](https://tailwindcss.com/docs/margin).

## Config
The `tailwind.config.js` consits of a basic set (76kB) of utility classes.
It is based on our Global Design Tokens and there should never be component based design tokens in it.

## Angular Integration
To integrate Tailwind into angular without ejecting it, i was following this article [UPDATED: Angular 11 and TailwindCSS 2](https://jacobneterer.medium.com/angular-and-tailwindcss-2388fb6e0bab)
which basically broke the whole angular setup:
* no `ng test`
* no `ng generate`

This article [Angular 11 and TailwindCSS with HMR](https://lhargil.com/angular-11-and-tailwindcss-with-hmr/) came to help and
now everything works as expected.
